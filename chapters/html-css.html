HTML & CSS

1. What is Semantic HTML?

Basic
Simple Explanation
Using HTML tags that describe their meaning (e.g., <header>, <article>) instead of generic <div>.
Detailed Answer

Semantic HTML improves accessibility (screen readers), SEO, and code readability. Examples: <nav>, <main>, <footer>, <section>, <article>.

<!-- Bad -->
<div class="header"></div>

<!-- Good -->
<header></header>

2. Explain the Box Model.

Basic
Simple Explanation
Every element is a box with: Content → Padding → Border → Margin (from inside out).
Detailed Answer

By default, width and height only apply to the content. Use box-sizing: border-box to include padding and border in the width/height calculation.

/* Standard Box Model */
.box {
  width: 100px;
  padding: 10px;
  border: 5px;
  /* Actual width = 100 + 20 + 10 = 130px */
}

/* Border Box Model (Recommended) */
.box {
  box-sizing: border-box;
  width: 100px;
  padding: 10px;
  border: 5px;
  /* Actual width = 100px (padding and border included) */
}

3. What is Flexbox and when to use it?

Intermediate
Simple Explanation
A layout model for arranging items in a row or column with flexible sizing.
Detailed Answer

Use Flexbox for one-dimensional layouts (either row or column). Key properties: justify-content (main axis), align-items (cross axis), flex-grow, flex-shrink.

.container {
  display: flex;
  justify-content: space-between;
  align-items: center;
}

.item {
  flex: 1; /* Grow to fill available space */
}

4. What is CSS Grid and when to use it?

Intermediate
Simple Explanation
A layout system for creating two-dimensional grids (rows AND columns).
Detailed Answer

Use Grid for complex layouts. Flexbox is for one-dimensional layouts, Grid is for two-dimensional. Key properties: grid-template-columns, grid-template-rows, gap.

.container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 20px;
}

/* Responsive Grid */
.container {
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
}

5. Difference between `position: relative`, `absolute`, `fixed`, and `sticky`?

Intermediate
Simple Explanation
Relative: Positioned relative to itself. Absolute: Positioned relative to nearest positioned ancestor. Fixed: Positioned relative to viewport. Sticky: Toggles between relative and fixed.
Detailed Answer
  • relative: Offset from its normal position, but space is reserved.
  • absolute: Removed from flow, positioned relative to nearest positioned ancestor.
  • fixed: Removed from flow, stays in same place when scrolling.
  • sticky: Acts like relative until scroll threshold, then becomes fixed.
.sticky-header {
  position: sticky;
  top: 0; /* Sticks to top when scrolling */
}

6. What is Specificity in CSS?

Intermediate
Simple Explanation
The algorithm that determines which CSS rule wins when multiple rules target the same element.
Detailed Answer

Specificity hierarchy (highest to lowest):

  1. Inline styles (style="...")
  2. IDs (#id)
  3. Classes, attributes, pseudo-classes (.class, [type], :hover)
  4. Elements and pseudo-elements (div, ::before)

!important overrides everything (use sparingly).

/* Specificity: 0-0-1 */
div { color: red; }

/* Specificity: 0-1-0 (WINS) */
.text { color: blue; }

/* Specificity: 1-0-0 (WINS) */
#heading { color: green; }

7. What are Pseudo-classes and Pseudo-elements?

Basic
Simple Explanation
Pseudo-classes (single colon) select elements in a specific state. Pseudo-elements (double colon) style specific parts of an element.
Detailed Answer

Pseudo-classes: :hover, :focus, :nth-child(), :first-child

Pseudo-elements: ::before, ::after, ::first-letter, ::selection

/* Pseudo-class */
button:hover {
  background: blue;
}

/* Pseudo-element */
p::first-letter {
  font-size: 2em;
  color: red;
}

8. Explain CSS Animations vs Transitions.

Intermediate
Simple Explanation
Transitions: Smooth change from one state to another (e.g., hover). Animations: Multi-step sequences with keyframes.
Detailed Answer

Use transitions for simple state changes. Use animations for complex, multi-step effects.

/* Transition */
.button {
  transition: background 0.3s ease;
}
.button:hover {
  background: blue;
}

/* Animation */
@keyframes slideIn {
  from { transform: translateX(-100%); }
  to { transform: translateX(0); }
}

.element {
  animation: slideIn 0.5s ease-out;
}

9. What is the Critical Rendering Path?

Advanced
Simple Explanation
It's the sequence of steps the browser takes to convert HTML, CSS, and JS into actual pixels on the screen.
Detailed Answer

Optimizing the CRP is key to improving web performance (Core Web Vitals). The steps are:

  1. DOM Construction: Parse HTML into the DOM tree.
  2. CSSOM Construction: Parse CSS into the CSSOM tree.
  3. Render Tree: Combine DOM and CSSOM. (Elements with display: none are excluded).
  4. Layout (Reflow): Calculate geometry (position and size) of each node.
  5. Paint: Fill in pixels (colors, borders, shadows).
  6. Composite: Layering the painted parts together.

10. What is BEM and why use it?

Intermediate
Simple Explanation
BEM stands for Block, Element, Modifier. It's a naming convention for CSS classes to make code reusable and maintainable.
Detailed Answer
  • Block: Standalone entity (e.g., .card).
  • Element: Part of a block (e.g., .card__title).
  • Modifier: A flag on a block or element (e.g., .card--featured).

It prevents style leaks and specificity wars by keeping specificity low (mostly single class selectors).

/* Block */
.btn {}

/* Element */
.btn__icon {}

/* Modifier */
.btn--primary {}
.btn--large {}

11. How does z-index and Stacking Context work?

Advanced
Simple Explanation
z-index decides which element is "on top". But it only works if the element has a position (relative/absolute/fixed).
Detailed Answer

The Trap: A child with z-index: 999 can still be behind a neighbor with z-index: 1 if the child's parent has a lower z-index than the neighbor. This is called a Stacking Context.

.parent-1 { z-index: 1; position: relative; }
.child-999 { z-index: 999; position: absolute; }

.parent-2 { z-index: 2; position: relative; }

/* .child-999 will be BEHIND .parent-2 
   because .parent-1 (z-index: 1) is below .parent-2 (z-index: 2). 
   The child cannot escape its parent's level. */

12. What are CSS Custom Properties (Variables)?

Intermediate
Simple Explanation
Variables you can use directly in CSS. They are great for theming (Dark Mode).
Detailed Answer

You define them with --name and use them with var(--name). Unlike SASS variables, they can be updated by JavaScript at runtime.

:root {
  --primary: blue;
}

button {
  background: var(--primary);
}

/* Dark Mode */
body.dark {
  --primary: lightblue; /* Automatically updates all buttons */
}

13. What are ARIA roles and when should you use them?

Advanced
Simple Explanation
Attributes that help screen readers understand custom controls. Rule #1: Don't use ARIA if a native HTML tag works.
Detailed Answer

Use them only when you are building complex widgets (like a custom dropdown or modal) that don't have a built-in HTML tag.

<!-- BAD: Re-inventing the wheel -->
<div role="button" onclick="...">Click</div>

<!-- GOOD: Using native element -->
<button onclick="...">Click</button>

<!-- GOOD: ARIA for custom widget -->
<div role="dialog" aria-labelledby="modal-title">
  <h2 id="modal-title">Settings</h2>
</div>

14. Pros and Cons of CSS-in-JS?

Advanced
Simple Explanation
Writing CSS inside JavaScript files (e.g., Styled Components). It solves "global namespace" issues but adds file size.
Detailed Answer

Pros: No class name clashes, easy dynamic styling based on props.
Cons: Slower performance (runtime parsing), larger JS bundle. 

// Styled Components Example
const Button = styled.button`
  background: ${props => props.primary ? 'blue' : 'gray'};
  color: white;
  padding: 10px;
`;

// Usage
<Button primary>Click Me</Button>

15. Difference between `display: none` and `visibility: hidden`?

Basic
Simple Explanation
display: none removes it completely (no space). visibility: hidden makes it invisible but it still takes up space.
Detailed Answer

Use display: none to completely hide something. Use visibility: hidden if you want to keep the layout structure stable but hide the content.

.removed {
  display: none; /* Element is GONE. Layout shifts. */
}

.invisible {
  visibility: hidden; /* Element is there, just transparent. No layout shift. */
}

16. Why is Tailwind CSS so popular?

2025 Ready
Simple Explanation
It lets you style elements by adding classes like flex, p-4, text-red-500 directly in HTML.
Detailed Answer

It speeds up development because you don't have to switch between HTML and CSS files or invent class names like .sidebar-wrapper-inner.

<!-- Traditional CSS -->
<button class="btn-primary">Click</button>

<!-- Tailwind CSS -->
<button class="bg-blue-500 text-white px-4 py-2 rounded hover:bg-blue-600">
  Click
</button>

17. What are CSS Container Queries?

2025 Ready
Simple Explanation
Like Media Queries, but they check the size of the parent container, not the whole screen.
Detailed Answer

This makes components truly reusable. A "Card" component can look one way in a wide sidebar and another way in a narrow column, regardless of the screen size.

.parent {
  container-type: inline-size;
}

.child {
  /* Default style */
  font-size: 1rem;
}

@container (min-width: 500px) {
  .child {
    /* Style when PARENT is wider than 500px */
    font-size: 2rem;
  }
}
