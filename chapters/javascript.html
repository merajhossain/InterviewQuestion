9. Explain Prototypal Inheritance.

Advanced
Simple Explanation
Objects in JavaScript have a link to a prototype object. When trying to access a property, JS checks the object first, then its prototype, and so on (Prototype Chain).
Detailed Answer

Every function has a prototype property. When you create an object using new Func(), the object's internal [[Prototype]] (exposed as __proto__) is linked to Func.prototype.

ES6 Classes are syntactic sugar over this system.

10. Key features introduced in ES6?

Basic
Simple Explanation
ES6 (ECMAScript 2015) was a major update. Key features include `let`/`const`, Arrow Functions, Template Literals, Destructuring, and Classes.
Detailed Answer
  • Block Scoping: `let` and `const`.
  • Arrow Functions: Concise syntax and lexical `this`.
  • Template Literals: String interpolation with backticks.
  • Destructuring: Extracting values from arrays/objects easily.
  • Default Parameters: `function(a = 1)`.
  • Spread/Rest Operator: `...args`.

11. Explain map(), filter(), and reduce().

Intermediate
Simple Explanation
Higher-order array methods. `map` transforms, `filter` selects, and `reduce` accumulates.
Detailed Answer
  • map: Returns a new array with the same length, where each element is transformed.
  • filter: Returns a new array with only elements that pass the condition.
  • reduce: Reduces the array to a single value (e.g., sum, object) based on an accumulator.
const nums = [1, 2, 3, 4];
const sum = nums.reduce((acc, curr) => acc + curr, 0); // 10

12. What is Event Bubbling and Capturing?

Intermediate
Simple Explanation
They describe the order in which event handlers are called when an event occurs on an element nested inside other elements.
Detailed Answer

Capturing Phase (Trickling): The event goes down from the window to the target element.

Target Phase: The event reaches the target.

Bubbling Phase: The event bubbles up from the target to the window.

By default, event listeners trigger in the Bubbling phase. Use addEventListener(type, listener, true) to listen in the Capturing phase.

13. Difference between Debounce and Throttle?

Advanced
Simple Explanation
Techniques to limit the rate at which a function is executed.
Detailed Answer

Debounce: Ensures that a function is not called until a certain amount of time has passed since the last time it was invoked. (e.g., "Search as you type" - wait until user stops typing).

Throttle: Ensures that a function is called at most once in a specified time period. (e.g., "Scroll event" - update only every 100ms).

14. Difference between Shallow and Deep Copy?

Intermediate
Simple Explanation
A shallow copy copies the object's top-level properties. Nested objects are copied as references. A deep copy duplicates everything, including nested objects.
Detailed Answer

Shallow Copy Methods: Object.assign({}, obj), Spread operator {...obj}.

Deep Copy Methods: JSON.parse(JSON.stringify(obj)) (has limitations with functions/undefined), or structuredClone(obj) (modern standard).

15. Difference between `==` and `===`?

Basic
Simple Explanation
`==` (Loose Equality) performs type coercion before comparison. `===` (Strict Equality) compares both value and type without coercion.
Detailed Answer

Always use === to avoid unexpected bugs caused by implicit type conversion.

5 == "5"  // true
5 === "5" // false
null == undefined // true
null === undefined // false

16. Difference between `null` and `undefined`?

Basic
Simple Explanation
`undefined` means a variable has been declared but not assigned a value. `null` is an assignment value that represents "no value" or "empty".
Detailed Answer

typeof undefined is "undefined". typeof null is "object" (a historical bug in JS).

17. What is Currying?

Advanced
Simple Explanation
Currying is the process of converting a function that takes multiple arguments into a sequence of functions that each take a single argument.
Detailed Answer

It helps in creating higher-order functions and partial application.

// Normal
function add(a, b) { return a + b; }

// Curried
function curriedAdd(a) {
  return function(b) {
    return a + b;
  };
}
curriedAdd(5)(10); // 15

18. What are Generators?

Advanced
Simple Explanation
Functions that can be paused and resumed. They are declared with `function*` and use `yield` to return values one by one.
Detailed Answer

When called, a generator returns an iterator object. You call .next() on the iterator to execute code until the next yield.

19. LocalStorage vs SessionStorage vs Cookies?

Intermediate
Simple Explanation
Different ways to store data on the client-side.
Detailed Answer
  • LocalStorage: Persists even after the browser is closed. Stores about 5-10MB.
  • SessionStorage: Persists only for the session (until the tab is closed). Stores about 5MB.
  • Cookies: Sent to the server with every HTTP request. Used for authentication. Stores 4KB.

20. How to select elements in the DOM?

Basic
Simple Explanation
Using methods like `getElementById`, `querySelector`, etc.
Detailed Answer
  • document.getElementById('id'): Fast, selects one element.
  • document.querySelector('.class'): Selects the first match (CSS selector syntax).
  • document.querySelectorAll('div'): Selects all matches (returns NodeList).

21. Common causes of Memory Leaks in JS?

Advanced
Simple Explanation
Memory leaks happen when the application holds references to objects that are no longer needed, preventing the Garbage Collector from freeing up memory.
Detailed Answer

Common causes:

  • Global variables: Accidentally creating globals.
  • Closures: Holding references to large scopes unnecessarily.
  • Event Listeners: Forgetting to remove listeners on unmount (especially in SPA).
  • Timers: Uncleared `setInterval`.

22. What are Service Workers?

Advanced
Simple Explanation
Scripts that run in the background, separate from the web page. They enable features like Offline support (PWA), Push Notifications, and Background Sync.
Detailed Answer

They act as a proxy between the web app and the network. They can intercept network requests and serve cached responses.

23. What are Web Workers?

Advanced
Simple Explanation
Web Workers allow you to run JavaScript code in the background, on a separate thread, to avoid blocking the main UI thread.
Detailed Answer

Useful for heavy computations (image processing, large data sorting) that would otherwise freeze the interface.

24. What is "use strict"?

Basic
Simple Explanation
A directive that enforces stricter parsing and error handling in JavaScript code.
Detailed Answer

It prevents common mistakes like using undeclared variables, duplicate parameter names, and using reserved keywords.

25. CommonJS vs ES Modules?

Intermediate
Simple Explanation
CommonJS: Node.js standard (require). ES Modules: Web standard (import).
Detailed Answer

ES Modules are statically analyzable (allowing Tree Shaking) and asynchronous. CommonJS is dynamic and synchronous.

// CommonJS (Node.js)
const fs = require('fs');
module.exports = function() {};

// ES Modules (Modern Web)
import fs from 'fs';
export default function() {};
