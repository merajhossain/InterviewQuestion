HTML & CSS

9. What is the Critical Rendering Path?

Advanced
Simple Explanation
It's the sequence of steps the browser takes to convert HTML, CSS, and JS into actual pixels on the screen.
Detailed Answer

Optimizing the CRP is key to improving web performance (Core Web Vitals). The steps are:

  1. DOM Construction: Parse HTML into the DOM tree.
  2. CSSOM Construction: Parse CSS into the CSSOM tree.
  3. Render Tree: Combine DOM and CSSOM. (Elements with display: none are excluded).
  4. Layout (Reflow): Calculate geometry (position and size) of each node.
  5. Paint: Fill in pixels (colors, borders, shadows).
  6. Composite: Layering the painted parts together.

10. What is BEM and why use it?

Intermediate
Simple Explanation
BEM stands for Block, Element, Modifier. It's a naming convention for CSS classes to make code reusable and maintainable.
Detailed Answer
  • Block: Standalone entity (e.g., .card).
  • Element: Part of a block (e.g., .card__title).
  • Modifier: A flag on a block or element (e.g., .card--featured).

It prevents style leaks and specificity wars by keeping specificity low (mostly single class selectors).

/* Block */
.btn {}

/* Element */
.btn__icon {}

/* Modifier */
.btn--primary {}
.btn--large {}

11. How does z-index and Stacking Context work?

Advanced
Simple Explanation
z-index decides which element is "on top". But it only works if the element has a position (relative/absolute/fixed).
Detailed Answer

The Trap: A child with z-index: 999 can still be behind a neighbor with z-index: 1 if the child's parent has a lower z-index than the neighbor. This is called a Stacking Context.

Code Example
.parent-1 { z-index: 1; position: relative; }
.child-999 { z-index: 999; position: absolute; }

.parent-2 { z-index: 2; position: relative; }

/* .child-999 will be BEHIND .parent-2 
   because .parent-1 (z-index: 1) is below .parent-2 (z-index: 2). 
   The child cannot escape its parent's level. */

12. What are CSS Custom Properties (Variables)?

Intermediate
Simple Explanation
Variables you can use directly in CSS. They are great for theming (Dark Mode).
Detailed Answer

You define them with --name and use them with var(--name). Unlike SASS variables, they can be updated by JavaScript at runtime.

Code Example
:root {
  --primary: blue;
}

button {
  background: var(--primary);
}

/* Dark Mode */
body.dark {
  --primary: lightblue; /* Automatically updates all buttons */
}

13. What are ARIA roles and when should you use them?

Advanced
Simple Explanation
Attributes that help screen readers understand custom controls. Rule #1: Don't use ARIA if a native HTML tag works.
Detailed Answer

Use them only when you are building complex widgets (like a custom dropdown or modal) that don't have a built-in HTML tag.

Code Example
<!-- BAD: Re-inventing the wheel -->
<div role="button" onclick="...">Click</div>

<!-- GOOD: Using native element -->
<button onclick="...">Click</button>

<!-- GOOD: ARIA for custom widget -->
<div role="dialog" aria-labelledby="modal-title">
  <h2 id="modal-title">Settings</h2>
</div>

14. Pros and Cons of CSS-in-JS?

Advanced
Simple Explanation
Writing CSS inside JavaScript files (e.g., Styled Components). It solves "global namespace" issues but adds file size.
Detailed Answer

Pros: No class name clashes, easy dynamic styling based on props.
Cons: Slower performance (runtime parsing), larger JS bundle.

Code Example

  background: ${props => props.primary ? 'blue' : 'gray'};
  color: white;
  padding: 10px;

// Usage
<Button primary>Click Me</Button>

15. Difference between `display: none` and `visibility: hidden`?

Basic
Simple Explanation
display: none removes it completely (no space). visibility: hidden makes it invisible but it still takes up space.
Detailed Answer

Use display: none to completely hide something. Use visibility: hidden if you want to keep the layout structure stable but hide the content.

Code Example
.removed {
  display: none; /* Element is GONE. Layout shifts. */
}

.invisible {
  visibility: hidden; /* Element is there, just transparent. No layout shift. */
}

17. What are CSS Container Queries?

2025 Ready
Simple Explanation
Like Media Queries, but they check the size of the parent container, not the whole screen.
Detailed Answer

This makes components truly reusable. A "Card" component can look one way in a wide sidebar and another way in a narrow column, regardless of the screen size.

Code Example
.parent {
  container-type: inline-size;
}

.child {
  /* Default style */
  font-size: 1rem;
}

@container (min-width: 500px) {
  .child {
    /* Style when PARENT is wider than 500px */
    font-size: 2rem;
  }
}

9. Explain Prototypal Inheritance.

Advanced
Simple Explanation
Objects in JavaScript have a link to a prototype object. When trying to access a property, JS checks the object first, then its prototype, and so on (Prototype Chain).
Detailed Answer

Every function has a prototype property. When you create an object using new Func(), the object's internal [[Prototype]] (exposed as __proto__) is linked to Func.prototype.

ES6 Classes are syntactic sugar over this system.

11. Explain map(), filter(), and reduce().

Intermediate
Simple Explanation
Higher-order array methods. `map` transforms, `filter` selects, and `reduce` accumulates.
Detailed Answer
  • map: Returns a new array with the same length, where each element is transformed.
  • filter: Returns a new array with only elements that pass the condition.
  • reduce: Reduces the array to a single value (e.g., sum, object) based on an accumulator.
const nums = [1, 2, 3, 4];
const sum = nums.reduce((acc, curr) => acc + curr, 0); // 10

12. What is Event Bubbling and Capturing?

Intermediate
Simple Explanation
They describe the order in which event handlers are called when an event occurs on an element nested inside other elements.
Detailed Answer

Capturing Phase (Trickling): The event goes down from the window to the target element.

Target Phase: The event reaches the target.

Bubbling Phase: The event bubbles up from the target to the window.

By default, event listeners trigger in the Bubbling phase. Use addEventListener(type, listener, true) to listen in the Capturing phase.

13. Difference between Debounce and Throttle?

Advanced
Simple Explanation
Techniques to limit the rate at which a function is executed.
Detailed Answer

Debounce: Ensures that a function is not called until a certain amount of time has passed since the last time it was invoked. (e.g., "Search as you type" - wait until user stops typing).

Throttle: Ensures that a function is called at most once in a specified time period. (e.g., "Scroll event" - update only every 100ms).

14. Difference between Shallow and Deep Copy?

Intermediate
Simple Explanation
A shallow copy copies the object's top-level properties. Nested objects are copied as references. A deep copy duplicates everything, including nested objects.
Detailed Answer

Shallow Copy Methods: Object.assign({}, obj), Spread operator {...obj}.

Deep Copy Methods: JSON.parse(JSON.stringify(obj)) (has limitations with functions/undefined), or structuredClone(obj) (modern standard).

15. Difference between `==` and `===`?

Basic
Simple Explanation
`==` (Loose Equality) performs type coercion before comparison. `===` (Strict Equality) compares both value and type without coercion.
Detailed Answer

Always use === to avoid unexpected bugs caused by implicit type conversion.

5 == "5"  // true
5 === "5" // false
null == undefined // true
null === undefined // false

16. Difference between `null` and `undefined`?

Basic
Simple Explanation
`undefined` means a variable has been declared but not assigned a value. `null` is an assignment value that represents "no value" or "empty".
Detailed Answer

typeof undefined is "undefined". typeof null is "object" (a historical bug in JS).

17. What is Currying?

Advanced
Simple Explanation
Currying is the process of converting a function that takes multiple arguments into a sequence of functions that each take a single argument.
Detailed Answer

It helps in creating higher-order functions and partial application.

// Normal
function add(a, b) { return a + b; }

// Curried
function curriedAdd(a) {
  return function(b) {
    return a + b;
  };
}
curriedAdd(5)(10); // 15

18. What are Generators?

Advanced
Simple Explanation
Functions that can be paused and resumed. They are declared with `function*` and use `yield` to return values one by one.
Detailed Answer

When called, a generator returns an iterator object. You call .next() on the iterator to execute code until the next yield.

19. LocalStorage vs SessionStorage vs Cookies?

Intermediate
Simple Explanation
Different ways to store data on the client-side.
Detailed Answer
  • LocalStorage: Persists even after the browser is closed. Stores about 5-10MB.
  • SessionStorage: Persists only for the session (until the tab is closed). Stores about 5MB.
  • Cookies: Sent to the server with every HTTP request. Used for authentication. Stores 4KB.

21. Common causes of Memory Leaks in JS?

Advanced
Simple Explanation
Memory leaks happen when the application holds references to objects that are no longer needed, preventing the Garbage Collector from freeing up memory.
Detailed Answer

Common causes:

  • Global variables: Accidentally creating globals.
  • Closures: Holding references to large scopes unnecessarily.
  • Event Listeners: Forgetting to remove listeners on unmount (especially in SPA).
  • Timers: Uncleared `setInterval`.

25. CommonJS vs ES Modules?

Intermediate
Simple Explanation
CommonJS: Node.js standard (require). ES Modules: Web standard (import).
Detailed Answer

ES Modules are statically analyzable (allowing Tree Shaking) and asynchronous. CommonJS is dynamic and synchronous.

module.exports = function() {};

import fs from 'fs';
export default function() {};

TypeScript

1. Difference between Interface and Type?

Basic
Simple Explanation
Interfaces are for defining object shapes and support declaration merging. Types are more flexible (unions, primitives, intersections).
Detailed Answer

Use interface for public API definitions (libraries) as they are cleaner and extendable. Use type for complex type compositions like unions or when you need to define a primitive alias.

// Interface (Open to extension)
interface User {
  name: string;
}
interface User { // Declaration Merging
  age: number;
}

// Type (Flexible)
type ID = string | number; // Union
type UserState = 'active' | 'inactive';

2. What are Generics?

Intermediate
Simple Explanation
"Variables" for types. They allow you to create reusable components that work with a variety of types while maintaining type safety.
Detailed Answer

Generics capture the type provided by the user (e.g., <T>) so it can be used later in the function signature or class. This avoids using any and losing type information.

function identity<T>(arg: T): T {
  return arg;
}

const num = identity(42); // T is inferred as number
const str = identity("Hello"); // T is inferred as string

3. `any` vs `unknown`?

Intermediate
Simple Explanation
any: "I don't care about the type" (disables type checking). unknown: "I don't know the type yet" (must check before using).
Detailed Answer

unknown is the type-safe counterpart of any. You cannot perform operations on an unknown value without first narrowing its type (e.g., using typeof).

let a: any = 10;
a.foo(); // No Error (Runtime Crash)

let u: unknown = 10;
// u.foo(); // Error: Object is of type 'unknown'

if (typeof u === 'number') {
  console.log(u.toFixed(2)); // OK
}

4. Union (`|`) vs Intersection (`&`) Types?

Basic
Simple Explanation
Union: A value can be one of several types (OR). Intersection: A value must satisfy multiple types (AND).
Detailed Answer

Unions are used for values that can vary (e.g., string | null). Intersections are used to combine existing types into a new one.

// Union (OR)
type Status = 'success' | 'error';

// Intersection (AND)
type Draggable = { drag: () => void };
type Resizable = { resize: () => void };
type Widget = Draggable & Resizable; // Must have both methods

5. Explain `keyof` and `typeof`.

Intermediate
Simple Explanation
typeof: Gets the type of a value/variable. keyof: Gets a union of all keys (property names) of a type.
Detailed Answer

They are often used together to create dynamic types based on existing objects.

const user = { name: 'Alice', age: 30 };

type User = typeof user; // { name: string; age: number }
type UserKeys = keyof User; // "name" | "age"

6. Common Utility Types (Pick, Omit, Partial)?

Intermediate
Simple Explanation
Built-in tools to transform types. Partial: Makes all props optional. Pick: Selects specific props. Omit: Removes specific props.
Detailed Answer

These help avoid code duplication by deriving new types from existing ones.

interface Todo {
  title: string;
  desc: string;
  completed: boolean;
}

type MiniTodo = Pick<Todo, 'title' | 'completed'>;
type NewTodo = Omit<Todo, 'completed'>;
type UpdateTodo = Partial<Todo>; // All optional

7. What are Type Guards?

Advanced
Simple Explanation
Techniques to narrow down the type of a variable within a conditional block.
Detailed Answer

Common guards include typeof, instanceof, and custom "type predicate" functions (functions returning arg is Type).

function isString(val: unknown): val is string {
  return typeof val === 'string';
}

function process(val: unknown) {
  if (isString(val)) {
    console.log(val.toUpperCase()); // TS knows val is string here
  }
}

8. What is a Discriminated Union?

Advanced
Simple Explanation
A pattern where every type in a union has a common literal property (the "tag") which TypeScript uses to narrow the type.
Detailed Answer

This is the standard way to handle polymorphic data in Redux actions or state management.

interface Circle { kind: 'circle'; radius: number; }
interface Square { kind: 'square'; side: number; }
type Shape = Circle | Square;

function getArea(s: Shape) {
  if (s.kind === 'circle') {
    return Math.PI * s.radius ** 2; // TS knows it's Circle
  }
  return s.side * s.side; // TS knows it's Square
}

9. What are Mapped Types?

Advanced
Simple Explanation
A way to create a new type by iterating over the keys of an existing type.
Detailed Answer

They are like .map() for types. You can add/remove modifiers like readonly or ? (optional).

type ReadOnly<T> = {
  readonly [K in keyof T]: T[K];
};

type User = { name: string; age: number };
type ReadOnlyUser = ReadOnly<User>; 
// { readonly name: string; readonly age: number; }

10. What is a `const` assertion?

Intermediate
Simple Explanation
A way to tell TypeScript that an expression should be treated as a literal, immutable value, not a general type.
Detailed Answer

It makes arrays readonly tuples and object properties readonly literals.

const config = {
  endpoint: 'https://api.com',
  retries: 3
} as const;

// config.endpoint is type "https://api.com" (not string)
// config.retries is type 3 (not number)
// config is readonly

React

1. What is the Virtual DOM?

Basic
Simple Explanation
A lightweight copy of the actual DOM in memory. React uses it to optimize updates.
Detailed Answer

When state changes, React creates a new Virtual DOM tree. It compares this new tree with the previous one (Diffing Algorithm) and calculates the minimum number of changes required to update the real DOM (Reconciliation).

// React Element (Virtual DOM)
const element = {
  type: 'h1',
  props: { children: 'Hello World' }
};

// React compares this object with the previous one
// and updates the real DOM only if they differ.

2. What is JSX?

Basic
Simple Explanation
JSX (JavaScript XML) is a syntax extension for JavaScript that allows you to write HTML-like code inside JavaScript.
Detailed Answer

Browsers cannot read JSX. It must be transpiled (usually by Babel) into standard JavaScript React.createElement() calls.

// JSX
const element = <h1>Hello</h1>;

// Transpiled to JS
const element = React.createElement('h1', null, 'Hello');

3. Difference between Class and Functional Components?

Basic
Simple Explanation
Class: Uses ES6 classes, `this`, and lifecycle methods. Functional: Plain functions using Hooks for state/effects.
Detailed Answer

Functional components are the modern standard. They are more concise, easier to test, and avoid common pitfalls with `this` binding.

// Class Component
class Welcome extends React.Component {
  render() { return <h1>Hello, {this.props.name}</h1>; }
}

// Functional Component
function Welcome(props) {
  return <h1>Hello, {props.name}</h1>;
}

4. Difference between State and Props?

Basic
Simple Explanation
Props: Read-only data passed into a component (like function arguments). State: Mutable data managed within a component (like local variables).
Detailed Answer

Props allow parent-to-child communication. State allows a component to change its own data over time and trigger re-renders.

// Parent passes props
<User name="Alice" />

// Child uses props and manages own state
function User(props) {
  const [active, setActive] = useState(false); // State
  return (
    <div>
      {props.name} is {active ? 'Active' : 'Inactive'}
    </div>
  );
}

5. What are the React Lifecycle Phases?

Intermediate
Simple Explanation
Mounting (Birth), Updating (Growth), and Unmounting (Death).
Detailed Answer

In functional components, we use the `useEffect` hook to handle these phases.

  • Mounting: useEffect(..., [])
  • Updating: useEffect(..., [dep])
  • Unmounting: useEffect(() => cleanup, ...)
useEffect(() => {
  console.log("Mounted");

  return () => {
    console.log("Unmounted");
  };
}, []); // Empty array = run once on mount

6. Explain the useState Hook.

Basic
Simple Explanation
A Hook that lets you add React state to functional components.
Detailed Answer

It returns a pair: the current state value and a function that lets you update it. Updating state triggers a re-render.

const [count, setCount] = useState(0);

// Update state (triggers re-render)
setCount(count + 1);

// Functional update (safer for multiple updates)
setCount(prev => prev + 1);

7. Explain the useEffect Hook.

Intermediate
Simple Explanation
Used for side effects: fetching data, directly updating the DOM, and timers.
Detailed Answer

It runs after every render by default. You can control when it runs using the dependency array. 

useEffect(() => {
  // Runs on mount + when 'userId' changes
  fetchData(userId);
}, [userId]); 

useEffect(() => {
  // Runs only on mount
}, []);

8. What is the Context API?

Intermediate
Simple Explanation
A way to share values (global state) between components without explicitly passing props through every level of the tree (Prop Drilling).
Detailed Answer

Commonly used for theme, current user, or language settings.

// 1. Create Context
const ThemeContext = createContext('light');

// 2. Provide Value
<ThemeContext.Provider value="dark">
  <Toolbar />
</ThemeContext.Provider>

// 3. Consume Value
const theme = useContext(ThemeContext); // "dark"

9. What is useRef used for?

Intermediate
Simple Explanation
Used to access DOM elements directly OR to store a mutable value that persists across renders without causing a re-render.
Detailed Answer

It returns a mutable ref object whose .current property is initialized to the passed argument.

const inputRef = useRef(null);

const focusInput = () => {
  // Access DOM node directly
  inputRef.current.focus();
};

return <input ref={inputRef} />;

10. Difference between useMemo and useCallback?

Advanced
Simple Explanation
useMemo: Caches a calculated value. useCallback: Caches a function definition.
Detailed Answer

Both are used for performance optimization to prevent unnecessary re-computations or re-renders of child components (when used with React.memo).

// Returns a memoized value
const expensiveValue = useMemo(() => compute(a, b), [a, b]);

// Returns a memoized function
const handleClick = useCallback(() => {
  doSomething(a);
}, [a]);

11. What are Custom Hooks?

Intermediate
Simple Explanation
JavaScript functions that start with "use" and can call other hooks. They allow you to extract and reuse stateful logic between components.
Detailed Answer

They follow the same rules as standard hooks. Common use cases include fetching data, handling local storage, or managing window events.

// Custom Hook
function useWindowWidth() {
  const [width, setWidth] = useState(window.innerWidth);
  useEffect(() => {
    const handleResize = () => setWidth(window.innerWidth);
    window.addEventListener('resize', handleResize);
    return () => window.removeEventListener('resize', handleResize);
  }, []);
  return width;
}

// Usage
const width = useWindowWidth();

12. When to use Context API vs Redux?

Intermediate
Simple Explanation
Context: Low-frequency updates (Theme, User, Locale). Built-in. Redux: High-frequency updates, complex state logic, or when you need middleware/devtools.
Detailed Answer

Context can cause performance issues if not used carefully (re-renders all consumers). Redux is optimized for performance and scalability.

13. What is Prop Drilling?

Basic
Simple Explanation
The process of passing data from a parent component down to a deep child component through intermediate components that do not need the data.
Detailed Answer

It makes code harder to maintain. Solved by using Context API, Redux, or Component Composition. 

// Prop Drilling
<App user={user} />
  <Layout user={user} /> // Doesn't use user
    <Header user={user} /> // Doesn't use user
      <Avatar user={user} /> // Finally uses user

14. What are Error Boundaries?

Intermediate
Simple Explanation
React components that catch JavaScript errors anywhere in their child component tree, log those errors, and display a fallback UI.
Detailed Answer

They must be Class Components implementing static getDerivedStateFromError() or componentDidCatch(). They do not catch errors in event handlers or async code.

class ErrorBoundary extends React.Component {
  state = { hasError: false };
  static getDerivedStateFromError(error) {
    return { hasError: true };
  }
  render() {
    if (this.state.hasError) return <h1>Something went wrong.</h1>;
    return this.props.children;
  }
}

A way to group a list of children without adding extra nodes (like extra `div`s) to the DOM.

Useful for returning multiple elements from a component or maintaining valid HTML structure (e.g., in tables or lists).

16. What are Portals?

Intermediate
Simple Explanation
A way to render children into a DOM node that exists outside the DOM hierarchy of the parent component.
Detailed Answer

Commonly used for modals, tooltips, and popups to avoid z-index or overflow issues.

ReactDOM.createPortal(

17. Controlled vs Uncontrolled Components?

Intermediate
Simple Explanation
Controlled: Form data is handled by React state. Uncontrolled: Form data is handled by the DOM itself (using refs).
Detailed Answer

Controlled components are recommended as they support instant validation and conditional disabling. Uncontrolled are closer to traditional HTML.

// Controlled
<input value={name} onChange={e => setName(e.target.value)} />

// Uncontrolled
<input ref={inputRef} />

18. What is a Higher-Order Component (HOC)?

Advanced
Simple Explanation
A function that takes a component and returns a new component with enhanced functionality.
Detailed Answer

It's a pattern for reusing component logic (e.g., authentication, logging). Hooks have largely replaced HOCs.

function withAuth(Component) {
  return function WrappedComponent(props) {
    if (!props.isAuthenticated) return <Login />;
    return <Component {...props} />;
  }
}

Detailed Answer

It performs a shallow comparison of props. You can provide a custom comparison function as a second argument.

const MyComponent = React.memo(function MyComponent(props) {

21. Explain the useReducer Hook.

Intermediate
Simple Explanation
An alternative to `useState` for managing complex state logic involving multiple sub-values or when the next state depends on the previous one.
Detailed Answer

It works similarly to Redux reducers: (state, action) => newState.

const [state, dispatch] = useReducer(reducer, initialState);

dispatch({ type: 'increment' });

22. Difference between SSR and CSR?

Advanced
Simple Explanation
CSR (Client-Side Rendering): Browser downloads empty HTML + JS bundle, then renders content. SSR (Server-Side Rendering): Server sends fully rendered HTML.
Detailed Answer

SSR is better for SEO and initial load performance (FCP). CSR provides a smoother interaction after the initial load.

// CSR (Standard React)
<div id="root"></div>
<script src="bundle.js"></script> // Renders content later

// SSR (Next.js / Remix)
<div id="root"><h1>Hello</h1></div> // Content arrives immediately

23. Why use Next.js?

Basic
Simple Explanation
Next.js is a React framework that provides SSR, Static Site Generation (SSG), file-based routing, and API routes out of the box.
Detailed Answer

It solves the configuration headache of setting up SSR manually with React and provides optimizations like Image and Font optimization.

Redux & State Management

1. What is Flux Architecture?

Basic
Simple Explanation
An application architecture for building client-side web apps. It enforces a unidirectional data flow.
Detailed Answer

Action -> Dispatcher -> Store -> View. Redux is a library inspired by Flux but simplifies it (single store, no dispatcher).

// Flux Flow
Action -> Dispatcher -> Store -> View -> Action

// Redux Flow
Action -> Reducer -> Store -> View -> Action

2. What are the Three Principles of Redux?

Basic
Simple Explanation
Single Source of Truth, State is Read-Only, Changes are made with Pure Functions (Reducers).
Detailed Answer
  1. Single Source of Truth: The state of your whole application is stored in an object tree within a single store.
  2. State is Read-Only: The only way to change the state is to emit an action.
  3. Changes are made with Pure Functions: To specify how the state tree is transformed by actions, you write pure reducers.

3. Explain Actions and Reducers.

Basic
Simple Explanation
Action: A plain object describing what happened. Reducer: A pure function describing how the state changes in response to an action.
Detailed Answer

Reducer signature: (previousState, action) => newState. It must never mutate the state directly.

// Action
const increment = { type: 'INCREMENT' };

// Reducer
function counter(state = 0, action) {
  switch (action.type) {
    case 'INCREMENT':
      return state + 1;
    default:
      return state;
  }
}

4. What is the Store?

Basic
Simple Explanation
The object that holds the application state.
Detailed Answer

It provides methods to access state (getState()), update state (dispatch(action)), and register listeners (subscribe(listener)).

import { createStore } from 'redux';
const store = createStore(counter);

store.subscribe(() => console.log(store.getState()));

store.dispatch({ type: 'INCREMENT' }); // 1

5. Redux Thunk vs Redux Saga?

Intermediate
Simple Explanation
Middleware for handling side effects (async logic). Thunk allows dispatching functions. Saga uses Generators to handle complex async flows.
Detailed Answer
  • Thunk: Simple, easy to learn. Good for simple async calls.
  • Saga: More powerful, testable, and better for complex flows (cancellation, debouncing, race conditions). Uses ES6 Generators.
// Thunk
const fetchUser = () => async (dispatch) => {
  const user = await api.getUser();
  dispatch({ type: 'USER_LOADED', payload: user });
};

// Saga
function* fetchUser() {
  const user = yield call(api.getUser);
  yield put({ type: 'USER_LOADED', payload: user });
}

6. `connect` HOC vs `useSelector`/`useDispatch` Hooks?

Intermediate
Simple Explanation
connect is the legacy way (HOC) to connect components to Redux. Hooks are the modern, recommended way.
Detailed Answer

Hooks are cleaner and reduce boilerplate. useSelector extracts data from the store state. useDispatch returns the dispatch function.

// Hooks (Modern)
const count = useSelector(state => state.counter);
const dispatch = useDispatch();

// connect (Legacy)
export default connect(mapStateToProps, mapDispatchToProps)(Component);

7. What is Redux Toolkit (RTK)?

Intermediate
Simple Explanation
The official, opinionated toolset for efficient Redux development. It simplifies store setup and reduces boilerplate.
Detailed Answer

It includes configureStore, createSlice (auto-generates actions/reducers), and uses Immer library internally to allow "mutating" logic in reducers (which Immer converts to immutable updates).

const counterSlice = createSlice({
  name: 'counter',
  initialState: 0,
  reducers: {
    increment: state => state + 1, // "Mutation" allowed thanks to Immer
  },
});

const store = configureStore({ reducer: counterSlice.reducer });

8. Why is Immutability important in Redux?

Intermediate
Simple Explanation
Redux relies on shallow equality checking to determine if the state has changed.
Detailed Answer

If you mutate the state directly, Redux won't detect the change, and components won't re-render. It also enables features like Time Travel Debugging.

// Bad (Mutation)
state.value = 1;
return state;

// Good (Immutable Update)
return { ...state, value: 1 };

9. What is Reselect?

Advanced
Simple Explanation
A library for creating memoized "selector" functions.
Detailed Answer

Selectors compute derived data from the Redux store. Reselect memoizes them so they are only recomputed when their input arguments change, improving performance.

const selectTax = createSelector(
  state => state.subtotal,
  state => state.taxPercent,
  (subtotal, taxPercent) => subtotal * (taxPercent / 100)
);

Browser Storage & Security

1. LocalStorage vs SessionStorage vs Cookies?

Intermediate
Simple Explanation
LocalStorage: Persists forever (5-10MB). SessionStorage: Persists for tab session (5MB). Cookies: Sent with requests (4KB).
Detailed Answer

Cookies are primarily for server-side reading (auth tokens). Storage APIs are for client-side data. Cookies have security flags (HttpOnly, Secure) that storage APIs lack.

// LocalStorage
localStorage.setItem('key', 'value');
localStorage.getItem('key');

// SessionStorage
sessionStorage.setItem('key', 'value');

// Cookies (Raw JS)
document.cookie = "username=John Doe; expires=Thu, 18 Dec 2025 12:00:00 UTC; path=/";

2. What is IndexedDB?

Advanced
Simple Explanation
A low-level API for client-side storage of significant amounts of structured data, including files/blobs.
Detailed Answer

Unlike LocalStorage (synchronous, string-only), IndexedDB is asynchronous, transactional, and can store objects. It's ideal for offline apps (PWAs) and caching large datasets.

const request = indexedDB.open("MyDatabase", 1);

request.onupgradeneeded = (event) => {
  const db = event.target.result;
  const objectStore = db.createObjectStore("customers", { keyPath: "ssn" });
  objectStore.createIndex("name", "name", { unique: false });
};

3. Session-based vs Token-based Authentication?

Intermediate
Simple Explanation
Session: Server stores state (ID in cookie). Token (JWT): Stateless (data in token).
Detailed Answer

Sessions are easier to invalidate but require server memory/DB lookups. Tokens scale better (no server state) but are harder to invalidate (require blocklists or short expiry).

// Token (JWT) Header
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

// Session Cookie
Cookie: session_id=1234567890abcdef;

4. Explain HttpOnly, Secure, and SameSite cookies.

Advanced
Simple Explanation
Security attributes to protect cookies from theft and misuse.
Detailed Answer
  • HttpOnly: Cannot be accessed via JavaScript (prevents XSS theft).
  • Secure: Only sent over HTTPS.
  • SameSite: Controls when cookies are sent with cross-site requests (Strict/Lax/None) to prevent CSRF.
Set-Cookie: id=a3fWa; Expires=Wed, 21 Oct 2025 07:28:00 GMT; Secure; HttpOnly; SameSite=Strict

5. What is Content Security Policy (CSP)?

Advanced
Simple Explanation
A security layer that helps detect and mitigate certain types of attacks, including Cross-Site Scripting (XSS) and data injection attacks.
Detailed Answer

It allows you to define which dynamic resources are allowed to load (e.g., "only allow scripts from my domain and google.com"). Defined via HTTP header or <meta> tag.

Content-Security-Policy: default-src 'self'; img-src *; media-src media1.com media2.com; script-src userscripts.example.com

6. What is the Same-Origin Policy?

Basic
Simple Explanation
A browser security mechanism that restricts how a document or script loaded from one origin can interact with a resource from another origin.
Detailed Answer

Two URLs have the same origin if the protocol, port (if specified), and host are the same. CORS is the way to relax this policy safely.

// Origin: http://store.company.com

http://store.company.com/dir/page.html // Success
http://store.company.com/dir/other.html // Success
https://store.company.com/page.html // Failure (Different Protocol)
http://store.company.com:81/dir/etc.html // Failure (Different Port)
http://news.company.com/dir/other.html // Failure (Different Host)

7. How do browser storage quotas work?

Advanced
Simple Explanation
Browsers limit how much data a site can store. It varies by device and available disk space.
Detailed Answer

If the quota is exceeded, the browser may throw a QuotaExceededError. You can check available space using navigator.storage.estimate(). Browsers may also evict data under storage pressure (LRU policy).

if (navigator.storage && navigator.storage.estimate) {
  const quota = await navigator.storage.estimate();
  // quota.usage -> used bytes
  // quota.quota -> available bytes
  console.log(`Using ${quota.usage} out of ${quota.quota} bytes.`);
}

8. Quick recap: XSS vs CSRF?

Intermediate
Simple Explanation
XSS: Attacker runs script on your site (steals data). CSRF: Attacker tricks user into doing action on your site (uses user's auth).
Detailed Answer

XSS is prevented by sanitization/CSP. CSRF is prevented by Anti-CSRF tokens and SameSite cookies.

Performance & Optimization

1. How do you optimize API calls?

Advanced
Simple Explanation
Caching, Debouncing/Throttling, and avoiding waterfalls.
Detailed Answer
  • Caching: Use React Query or SWR to cache responses.
  • Debouncing: Delay API calls until user stops typing (search bars).
  • Parallelization: Use Promise.all instead of awaiting sequentially.
  • Keep-Alive: Use HTTP/2 to reuse connections.
// Parallelization
const [user, posts] = await Promise.all([
  fetch('/api/user'),

// Caching (React Query)

Advanced
Simple Explanation
Use React DevTools Profiler to find them. Fix with React.memo, useMemo, and useCallback.

// Prevent re-render if props haven't changed
const MemoizedComponent = React.memo(MyComponent);

// Prevent function recreation
const handleClick = useCallback(() => {
  console.log('Clicked');
}, []);

5. How do you cancel a pending API request?

Intermediate
Simple Explanation
Using the AbortController API.
Detailed Answer

Pass signal: controller.signal to fetch. Call controller.abort() in the cleanup function of useEffect to cancel the request if the component unmounts or dependencies change (preventing Race Conditions).

7. Debounce vs Throttle?

Intermediate
Simple Explanation
Debounce: Wait for silence (e.g., search input). Throttle: Limit execution rate (e.g., scroll event).
Detailed Answer

Debounce executes only after the user stops triggering the event for X ms. Throttle executes at most once every X ms, regardless of how many times the event is triggered.

// Debounce (Search)
// User types 'h', 'e', 'l', 'l', 'o' -> API called once after 'o'

// Throttle (Scroll)
// User scrolls for 5 seconds -> Event fires every 100ms (50 times)

  • Integration Tests (Middle): Test how components work together.

  await page.click('#submit');
  await expect(page).toHaveURL('/dashboard');
});

Session: Server stores state. Client sends cookie ID. JWT (JSON Web Token): Stateless. Client stores token (usually in memory or cookie) and sends it in header.
Detailed Answer

JWT is better for microservices and mobile apps. Session is simpler and easier to invalidate (logout) on the server side.

// JWT Header
Authorization: Bearer <token>

// Session Cookie
Cookie: sessionId=<id>

8. What is CORS?

Intermediate
Simple Explanation
Cross-Origin Resource Sharing. A security feature that restricts web pages from making requests to a different domain than the one that served the web page.
Detailed Answer

The server must send specific headers (Access-Control-Allow-Origin) to allow the browser to accept the response.

// Server Response Header
Access-Control-Allow-Origin: https://my-app.com
Access-Control-Allow-Methods: GET, POST, PUT

11. Edge Computing vs Cloud Functions?

2025 Ready
Simple Explanation
Cloud Functions: Run in centralized data centers (e.g., us-east-1). Edge Functions: Run on servers closest to the user (CDN nodes).
Detailed Answer

Edge functions have lower latency (no cold starts usually) but often have a more limited runtime environment (e.g., V8 isolate instead of full Node.js).

// Edge Middleware (Next.js)
import { NextResponse } from 'next/server';

export function middleware(request) {
  const country = request.geo.country;
  if (country === 'US') {
    return NextResponse.rewrite(new URL('/us-home', request.url));
  }
}
